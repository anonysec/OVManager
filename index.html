Features

Everything you need to manage VPN users

A clean dashboard with real-time stats, user management, node orchestration, and subscription-based access.

👥

User Management

Create, edit, and disable users. Set traffic limits, expiry dates, and max simultaneous logins per user.

🖥️

Multi-Node Support

Manage multiple OpenVPN nodes from one panel. Push user configs and limits across all nodes in parallel.

🔑

JWT Authentication

Role-based access with main_admin and admin roles. Rate-limited login with bcrypt password hashing.

📊

Live Metrics

Real-time traffic usage, active connections, and per-node health monitoring on the dashboard.

📱

Telegram Bot

Fully interactive bot for user management — search, edit, renew, download configs via inline buttons.

🔒

Secure by Design

TLS support via Let's Encrypt, parameterized queries throughout, input validation on all endpoints.

Architecture

How It Works

OVManager is the control panel. OVNode is the agent that runs on each VPN server.

1

Deploy OVManager

Run the panel via Docker Compose with a single command. The web UI is served on port 2095.

2

Deploy OVNode on Each Server

Install the node agent on each OpenVPN server. It communicates back to the panel over HTTP with an API key.

3

Manage from the Dashboard

Create users, set limits, monitor traffic — all from the panel. OVNode handles OpenVPN config generation and cert lifecycle automatically.

Node Agent

OVNode — The Node Agent

OVNode runs alongside OpenVPN on each server, handling PKI, user certs, CCD files, and usage tracking. It talks to OVManager via a sync API.

🛡️

Automatic PKI

Initializes CA, generates server certs, and manages client certificates — idempotent and Docker-volume persistent.

🔄

Sync API

Pull-based sync from panel to node. Supports user creation, status changes, usage limits, and OVPN config downloads.

📈

Usage Tracking

Tracks per-user traffic consumption and active sessions, reported back to the panel for real-time dashboards.
